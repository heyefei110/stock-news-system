# 股票新闻自动化系统架构设计

## 系统概述

自动化定时任务系统，每日采集指定股票的最新资讯，经处理后推送到微信个人号。

## 技术栈选型

### 核心框架
- **运行时**: Python 3.10+
- **定时调度**: APScheduler (支持 cron 表达式、持久化、重试机制)
- **HTTP 请求**: httpx (异步、高性能)
- **HTML 解析**: BeautifulSoup4 + lxml
- **数据处理**: pandas (去重、分类)
- **LLM 摘要**: Claude API (智能摘要和分类)

### 存储层
- **配置存储**: SQLite (股票列表、推送时间等)
- **日志存储**: SQLite + logging 模块
- **缓存**: Redis (可选，用于去重和速率限制)

### 推送层
- **微信推送**: WeChatFerry / GeweBot / Server 酱 (个人号推送方案)

### 监控告警
- **异常通知**: 邮件 + 微信双重告警
- **健康检查**: 定时任务状态监控

## 数据流程

1. **6:00** - 定时任务触发
2. **6:00-6:30** - 并行采集各平台数据
3. **6:30-7:00** - 数据清洗、去重
4. **7:00-7:30** - LLM 智能摘要和分类
5. **7:30-8:00** - 格式化推送内容
6. **8:00-8:30** - 推送到微信 (预留重试时间)
7. **8:30** - 任务完成，记录日志

## 错误处理策略

- **采集失败**: 单平台失败不影响其他平台，重试 3 次
- **推送失败**: 立即重试，最多 5 次，失败后发送告警
- **LLM 超时**: 降级到关键词提取摘要
- **系统异常**: 记录详细日志，发送告警通知

## 安全性考虑

- API 密钥存储在 `.env` 文件
- 数据库连接使用参数化查询
- 爬虫遵守 robots.txt
- 请求速率限制，避免被封禁
