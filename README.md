# 股票资讯自动化系统

自动化采集股票相关新闻资讯，经处理后推送到微信个人号的定时任务系统。

## 功能特性

- ✅ **多源采集**：支持新浪财经、东方财富、雪球、腾讯新闻等主流财经平台
- ✅ **智能处理**：数据清洗、SimHash 去重、LLM 智能摘要
- ✅ **微信推送**：支持 Server 酱、Gewebot、WeChatFerry 等多种推送方式
- ✅ **定时调度**：每日自动执行，确保 8:30 前完成推送
- ✅ **错误重试**：指数退避重试机制，异常情况告警通知
- ✅ **配置管理**：Web 界面管理股票列表和系统设置
- ✅ **日志查询**：完整的任务执行日志和推送记录

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env` 并填写配置：

```bash
cp .env.example .env
```

编辑 `.env` 文件：

```env
# Claude API (用于智能摘要)
ANTHROPIC_API_KEY=your_api_key

# 微信推送 (任选其一)
SERVERCHAN_SENDKEY=your_sendkey
# 或
GEWE_BOT_URL=http://localhost:9500
GEWE_BOT_TOKEN=your_token

# 系统设置
LOG_LEVEL=INFO
DATABASE_PATH=./data/stock_news.db
```

### 3. 初始化数据库

```bash
python -m config.database
```

### 4. 运行系统

#### 单次执行测试

```bash
python main.py run
```

#### 启动定时任务

```bash
python main.py schedule
```

#### 启动 Web 管理后台

```bash
python -m web.app
```

访问 http://localhost:8000 查看管理后台。

## 配置说明

### 微信推送方式

#### Server 酱 (推荐)
1. 访问 https://sct.ftqq.com/ 获取 SendKey
2. 在 `.env` 中配置 `SERVERCHAN_SENDKEY`

#### Gewebot
1. 部署 Gewebot 服务
2. 配置 `GEWE_BOT_URL` 和 `GEWE_BOT_TOKEN`

#### WeChatFerry
1. 运行 WeChatFerry 服务
2. 配置 `WCF_HOST` 和 `WCF_PORT`

### 定时任务时间

默认每日 6:00 执行，确保 8:30 前完成。可通过 Web 管理后台修改。

## 错误处理

- **采集失败**: 单平台失败不影响其他平台，自动重试 3 次
- **推送失败**: 立即重试，最多 5 次，失败后发送告警
- **LLM 超时**: 降级到关键词提取摘要
- **系统异常**: 记录详细日志，发送邮件/微信告警

## 日志查询

访问 `/logs` 查看任务执行日志和推送记录，支持按时间范围筛选。

## 技术栈

- **运行时**: Python 3.10+
- **定时调度**: APScheduler
- **HTTP 请求**: httpx (异步)
- **Web 框架**: FastAPI
- **模板引擎**: Jinja2
- **数据库**: SQLite
- **LLM**: Claude API
- **日志**: loguru

## 故障排除

### 未采集到新闻

1. 检查网络连接
2. 检查股票名称是否正确
3. 查看日志确认采集器是否正常工作

### 推送失败

1. 检查推送配置是否正确
2. Server 酱需确认 SendKey 有效
3. 查看错误日志获取详细信息

### LLM 摘要失败

1. 检查 `ANTHROPIC_API_KEY` 是否配置
2. 网络问题导致 API 超时
3. 系统会自动降级到关键词摘要
